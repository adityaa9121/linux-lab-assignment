C:\ and drivers (9p filesystem): report -999001 used inodes, a known
quirk since 9p bridges to NTFS and does not track native Linux inodes.

5. Storage Health Observations
- The Linux root filesystem (/dev/sdd) is extremely healthy: only 1%
  space used and 1% inode usage, with 955G free.
- The Windows C: drive is more utilized at 27%, but still has 350G free.
- No filesystem is near capacity for either space or inodes.
- The 9p-mounted Windows drive does not report standard inode stats,
  which is expected behavior for this filesystem type in WSL.

6. Recommendations for Improving Storage Utilization
- Monitor the C: drive (27% used) periodically since it holds the bulk
  of used space; consider cleanup of temporary/cache files if usage grows.
- Continue using the ext4 root filesystem for Linux workloads, as it
  has abundant space and inode headroom.
- Set up periodic monitoring (e.g., a cron job running df -h and df -i)
  to catch storage or inode exhaustion early, especially before deploying
  a new application server that may generate large volumes of files/logs.
- Avoid storing large numbers of very small files where possible, since
  inode exhaustion (not just space) can also cause failures.
